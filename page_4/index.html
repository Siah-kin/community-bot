The four features, in plain terms

1. Recorded Q&A runs

You ask the bot a question in Telegram and it answers from a curated knowledge base. Right now those runs are scripted tests. Each run is stored end to end: the question, the answer, and the checks that fired along the way. A person reviewed at least one full run.

2. Member trust scores

Each member has a number that moves with what they do. Confirmed help pushes it up. The score lives as a database row, so it can be checked, not a badge someone hands out.

3. Bug report credits

Report a real bug, an admin confirms it, and a credit lands under your name. The credit record exists today; payouts come later, after the security review passes.

4. Ticket escalation

Open a support ticket with the bot. If it can't fix your problem, it hands the ticket to a human and records that handoff. Tickets don't vanish into a queue.

For devs: a Telegram bot, a Python backend, Postgres for the records. The code is public on GitHub, linked in the footer.
