System Update

When Bonzi says done,
it means done.

Bonzi's earn mechanics only hold up if the system can tell real work from claimed work. That was the gap we closed. Here's what changed and why it matters for contributors.

The problem with self-reported work

Most task systems have the same blind spot. A contributor says they finished something. The system records it as done. Rewards go out. Later, it turns out the work was partial, broken, or never actually connected to anything.

In a normal job that's annoying. In a work-and-reward system running on a blockchain, it means people collect tokens for things that didn't happen. That corrodes trust fast — and once it starts, it's hard to stop.

Bonzi used to rely on a mix of automated checks, AI interpretation, and convention. A task could look green even when key checks had not truly run. Scripts could report success without touching the files they were supposed to verify. An agent that sounded confident could advance a task past gates that were designed to stop incomplete work.

That's not a bug in one place. It's a structural weakness: the verification layer trusted intent instead of evidence.

What we fixed, and how

The hardening work pushed truth out of memory and into artifacts. Instead of asking "did the agent say it ran the check?" the system now asks "is there a file, a row, or an exit code that proves it?"

Three layers of evidence

  • 1 Scripts verify reality. Verification commands run against live files and actual state — not against memory or cached snapshots. If the file changed, the check sees the new file. If the database has zero rows, the check fails.
  • 2 Artifacts preserve what happened. Results are written to disk as structured records. An agent can't revise what it reported. The artifact is what it is.
  • 3 Gates decide if the evidence is enough. Before a task advances, a gate reads the artifact and checks a set of concrete conditions. Confidence and phrasing don't influence this. The conditions either pass or they don't.

This matters because it removes judgment from the critical path. Gates don't need to understand what an agent meant. They check facts: did this command exit with code 0, does this row exist, is this file newer than this timestamp. That's it.

What this means for raid-and-earn

Raid-and-earn is only worth building if the earn part is tied to real outcomes. A raiding model where everyone earns the same flat amount just for showing up will be gamed immediately. But a model where earnings scale with verified contribution and degrade when evidence weakens — that's defensible.

Proof-backed rewards

With the hardened truth layer in place, Bonzi can now tie reward thresholds to artifact state rather than task labels. A task marked "done" with a full evidence chain earns differently from one that passed on the minimum. A contribution that holds up over time earns more than one that degrades at the next audit.

That's the earn mechanic we're building toward: rewards that are fair because they're grounded in something the system actually measured.

It also opens the door to status degradation. If a task was accepted but later fails a re-verification — because a dependency changed, a check now fails, or the integration broke — the system can lower its status. That's not a punishment. It's the difference between a snapshot and a living record.

In a reputation economy, a claim only holds as much weight as the evidence behind it. Bonzi's artifact layer is what makes that sentence operational rather than aspirational.

Hybrid vetting: machine facts, human judgment

Full automation is brittle. Full manual review doesn't scale. The right split is: let the machine verify what's objectively checkable, and bring humans in where judgment actually matters.

Bonzi is now closer to that line. The verification layer handles the facts side: command execution, file presence, database rows, artifact freshness, integration state. These aren't interpretation calls. Either the row exists or it doesn't.

Human review kicks in for the edge cases that machines can't resolve cleanly: policy exemptions, disputed evidence, unusual contribution types, cases where the spec was ambiguous. This keeps the system scalable without becoming sloppy. Reviewers spend time where their judgment adds value, not on facts the system can already check.

Reputation that holds up

The longer-term goal is portable contributor reputation: badges, claims, and signals that follow a contributor across raids and carry weight with other communities. That only works if the reputation is hard to fake.

Reputation built on self-reported work is just branding. Reputation built on artifact-backed results that passed real gates is a record.

— The design principle behind Bonzi's verification layer

When Bonzi issues a reputation signal — a claim that someone completed verifiable work, that their contribution held up under re-check, that they consistently pass at a certain standard — that signal comes from the artifact chain, not from a chat log.

That's what makes it worth putting onchain. An onchain claim tied to nothing is just a token. An onchain claim tied to a verifiable chain of artifacts is evidence.

There is a simpler way to say all of this. Online, nobody verifies claims one by one. People trust what they have repeated experience with: it behaves the same way every time, it says what it knows and what it doesn't, and when it gets something wrong, it corrects itself in the open.

That is the standard Bonzi is built against, and you can check it instead of taking it on faith. Answers draw on one record, so they stay consistent. Known limits are stated, not papered over. And when a check fails, the failure is written to an append-only log: the same standard the system applies to everyone else.

Why this is the foundation, not just a fix

None of the things we want to build — raid coordination, fair rewards, hybrid vetting, portable reputation — hold up without a reliable way to distinguish real work from appearances. That's not a nice-to-have. It's the load-bearing layer.

This hardening work is that layer. It's not internal cleanup. It's the reason everything else we're building can be trusted. When Bonzi says a task is done, it's because the repo, the checks, and the verification artifacts all agree. Not because an agent sounded convincing.
