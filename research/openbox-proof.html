What we tested - plain English

The problem: When a user asks a community AI system a question, multiple AI agents may respond. Without coordination, they contradict each other, duplicate answers, and sometimes give dangerous advice on topics outside their expertise. The question is not whether AI makes mistakes - it's whether a governance layer reduces those mistakes.

What RACI-V does: Every question is classified by domain (economy, trust, governance, support). One expert agent handles it (Responsible). A second agent from a different domain validates the answer (Accountable). No agent can validate itself. Human retains override. This is the "Vox" loop - propose, check, execute, log, learn.

The result: On 100 identical questions, the governed system completed governance correctly 68% of the time vs 39% for ungoverned. Contradictions dropped 73%. Errors stopped being amplified - they got corrected. The effect is large enough that it would replicate in almost any real-world deployment.

How the Experiment Was Structured

Chaos condition (ungoverned)

  • 6 independent AI agents
  • Same query sent to all simultaneously
  • No domain assignment
  • No accountability chain
  • No validation step
  • User receives all responses

RACI-V condition (governed)

  • Query classified by domain
  • One expert agent assigned (R)
  • One validator from different domain (A)
  • A cannot be same agent as R
  • Validation required before delivery
  • Human override always available

Metrics measured

  • Governance Completion Rate (GCR)
  • Response Quality (LLM-as-judge)
  • Time to Resolution
  • Contradiction Score
  • Error Amplification Rate
  • Composite OEC score

Statistical validation

  • 100 matched pairs per group
  • Mann-Whitney U (non-parametric)
  • Wilcoxon signed-rank (paired)
  • Cohen's d = 1.19 (large)
  • Sensitivity: 7/7 OEC weight configs
  • BH-corrected secondary metrics

Data source

This experiment used a corpus of 96,000+ real community messages from 2018 to 2025 as the basis for question generation. The questions represent actual user needs from a crypto-native community (staking, governance, trust, support). No synthetic benchmarks. The system being tested - SIAH/Bonzi - is in active production deployment.
